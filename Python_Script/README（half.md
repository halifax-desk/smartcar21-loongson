# Python 巡线检测

基于 OpenCV 的智能车赛道巡线算法，包含两种检测方案，均通过摄像头实时运行。

## 环境要求

- Python 3.x
- OpenCV (`opencv-python`)
- NumPy

```bash
pip install opencv-python numpy
```

## 文件说明

| 文件 | 算法 | 适用性 |
|------|------|--------|
| `2ROI_Scan.py` | 双 ROI 列扫描 | 基础可用算法，实测可用 |
| `Full_8Area.py` | 8 区域扫描（基础实现） | 仅作学习参考，不可用于实际项目 |

---

## 2ROI_Scan.py

**基础可用算法**，实测可以巡线，但效果不如全功能的 SJTU 8Area 算法。

**核心原理**：将摄像头画面缩放到 160×120，灰度二值化后，在两个 ROI 区域内逐行查找白像素的左右边界，进而定位赛道边线和中心线。

**两个 ROI**：

| ROI | 位置 | 宽度 | 用途 |
|-----|------|------|------|
| ROI1（蓝框） | 画面中心向上 30% 高度，水平居中 | 70% 画面宽度 | 检测前方赛道 |
| ROI2（绿框） | 画面下半部分 | 100% 画面宽度 | 检测近处赛道 |

**可调参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `THRESHOLD` | 120 | 二值化阈值 |
| `SCAN_STEP` | 3 | 列扫描间隔 |
| `ROI1_HEIGHT_RATIO` | 0.3 | ROI1 高度占比 |
| `ROI1_WIDTH_RATIO` | 0.7 | ROI1 宽度占比 |
| `ROI2_HEIGHT_RATIO` | 0.5 | ROI2 高度占比 |
| `ROI2_WIDTH_RATIO` | 1.0 | ROI2 宽度占比 |

**显示内容**：
- 蓝框：ROI1 区域
- 绿框：ROI2 区域
- 绿点：每行检测到的左右边线
- 红点：每行计算出的中心线
- 黄色高亮：ROI2 最底部白色赛道区域
- 白色竖线：画面中心参考线

**运行**：

```bash
python 2ROI_Scan.py
```

按 `ESC` 退出。

## Full_8Area.py

⚠️ **仅作学习参考，不可用于实际项目。** 此为基础算法实现，**没有滤波、没有补线**，实际赛道中会出现断线、抖动等问题。完整的 SJTU 8Area 算法包含更多功能（滤波、补线、PID 控制等），此处仅实现了核心的扫描骨架。

**核心原理**：对二值化后的 120×188 灰度图像，从底部向上逐行扫描，在每一行中从中间向两侧寻找黑白跳变点作为赛道的左右边界，从而计算出中心线。

**可调参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `image_h` / `image_w` | 120 / 188 | 图像尺寸 |
| `ROI_ENABLE` | `True` | 是否启用 ROI 区域限制 |
| `ROI_TOP` / `ROI_BOTTOM` | 0 / 100 | ROI 的起止行号 |
| `Otsu_ENABLE` | `False` | 是否使用大津法自动二值化 |
| `image_thereshold` | 60 | 固定二值化阈值（Otsu 关闭时生效） |
| `bin_jump_num` | 1 | 采样跳转步长 |

**显示内容**：
- 蓝色点：左边线检测结果
- 红色点：右边线检测结果
- 绿色点：中心线
- 灰色矩形框：ROI 区域（启用时）
- 左上角显示实时 FPS 和运行时间

**运行**：

```bash
python Full_8Area.py
```

按 `ESC` 退出。

## 两种算法对比

| 特性 | 2ROI_Scan | Full_8Area |
|------|-----------|------------|
| 图像尺寸 | 160×120 | 120×188 |
| 二值化方式 | 固定阈值 | 固定阈值 / 大津法 |
| 扫描策略 | 逐行扫白像素首尾 | 从中间向两侧扫黑白跳变 |
| ROI 数量 | 2 个（近 + 远） | 1 个（可选） |
| 滤波 / 补线 | 无 | 无 |
| 实际可用性 | 基础可用 | 仅学习参考 |
